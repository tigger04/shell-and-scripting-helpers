Findings

Finding 1 — Behavioural difference in blockchart (line 1058)

Severity: Smell (potential bug depending on usage)

The bc_empty variable at line 1013 is set to '\U2B1B' (a backslash escape for a Unicode character). The two output paths now handle this differently:

  • echo -ne path (line 1056): interprets \U2B1B as the Unicode character (black square).
  • printf -v ... '%s' path (line 1058): stores the literal string \U2B1B, not the Unicode character.

This means callers using -v VAR would get a different value than callers relying on stdout.

Recommended fix (choose one):

  1. Replace the escape with the literal Unicode character:

    local bc_empty='⬛'

  2. Use %b instead of %s in the printf -v call:

    printf -v "$_block_output_var" '%b' "$block_display"

Option 1 is preferred — it makes both paths identical without relying on escape interpretation.

Finding 2 — Stale nameref in commented-out sanitize (line 929)

Finding 3 — local re-declared inside loop (lines 386, 399)

Severity: Smell (cosmetic)

In trim_quotes and quote_quotes, local _tmp_val=... appears inside a while loop. In bash, local inside a loop redeclares on each iteration — this is harmless (it just reassigns) but technically redundant. local only needs to appear once per function scope.

Recommended fix: Move the local declaration before the loop:

local _tmp_val
while [ $# -gt 0 ]; do
   _tmp_val="${!1}"
   ...
